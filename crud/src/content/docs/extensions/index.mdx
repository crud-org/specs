An **"Extension Specification"** adds optional capabilities to the [CRUD API Base Specification](/base/) that Participants MAY adopt. Extensions define abstract rules that Protocol Specifications then map to their transport mechanisms.

Extensions MUST NOT contradict Base Specification rules. An extension's rules only apply to Participants that declare support for it — typically via the `crudExtensions` Metadata field defined in [Section 3.4.2 General metadata](/base/general/#342-general-metadata).

Extension Specifications follow the same [RFC 2119](https://datatracker.ietf.org/doc/html/rfc2119) keyword conventions and [SemVer](https://semver.org/) versioning as the Base Specification.
